Geofence detection — server-side algorithm (implement exactly)

Add to controllers/locationController.js (or utils) a proximity detector with these tunable constants (put them into config/constants.js):

const RADIUS_METERS = 50;       // enter threshold
const SUSTAIN_TIME_MS = 6000;   // must stay inside for 6s
const LEAVE_RADIUS = 80;        // leave threshold (larger than RADIUS_METERS)
const MIN_UPDATE_INTERVAL_MS = 3000; // ignore updates faster than 3s from same device


Implementation outline (server):

Maintain an in-memory map (per active trip) to track recent driver pings and state:

// Example structure
activeTrips[tripId] = {
  lastPosition: {lat,lng,timestamp},
  insideWindow: { stopId, enterTimestamps: [ts1, ts2, ...], arrivedMarked: false, leftMarked: false },
  currentStopIndex: Number
}


On each driver:location_update:

Throttle using MIN_UPDATE_INTERVAL_MS (drop too-frequent pings).

Lookup trip and next stop (by trip.currentStopIndex or DB).

Compute distance = distanceMeters(pos, nextStop).

If distance <= RADIUS_METERS:

Append timestamp to enterTimestamps.

If earliest timestamp is older than now - SUSTAIN_TIME_MS and arrivedMarked === false:

Create StopEvent in DB {tripId, stopId, status:'ARRIVED', timestamp, location, source:'auto'}.

Set arrivedMarked = true.

Emit trip:stop_arrived to trip_<tripId>.

Recompute ETAs and emit trip:eta_update.

Else (distance > RADIUS_METERS):

Clear enterTimestamps.

If arrivedMarked === true and distance >= LEAVE_RADIUS and leftMarked === false:

Create StopEvent status:'LEFT' and persist. Set leftMarked = true.

Increment currentStopIndex for the trip in both in-memory state and DB.

Emit trip:stop_left and trip:eta_update.

Persist activeTrips updates to DB when a trip ends or at safe checkpoints to allow recovery.

Note: Because this uses in-memory tracking for sustain time, implement a fallback: if server restarts, existing trips should still work: on next GPS update, recompute from DB last events and rebuild currentStopIndex & flags.

ETA update behavior

When ARRIVED is recorded for stop i, recalc ETA for stops i+1..n as:

ETA[nextStop] = lastArrivalTime + averageSegmentTime(nextStopIndex-1) * remainingSegments

For testing: default averageSegmentTime = 120 seconds unless historic data exists. Save per-route segment averages when multiple trips have been recorded.

Emit trip:eta_update to the trip room.
