Core features (end-to-end working)

Backend:

JWT auth (plain-text password verification) with POST /api/auth/login.

REST endpoints: routes for buses, routes, stops, trips, events; minimal admin-protected endpoints.

Socket.IO server with socket auth: expects auth:token from clients; validates JWT and attaches socket.user.

Server-side geofence detection on driver:location_update to auto-create StopEvent ARRIVED/LEFT with debounce/hysteresis constants (RADIUS_METERS=50, SUSTAIN_TIME_MS=6000, LEAVE_RADIUS=80, MIN_UPDATE_INTERVAL_MS=3000).

In-memory activeTrips tracking with DB persistence of events and trip currentStopIndex.

Simple ETA engine (default per-segment time 120s) and trip:eta_update emission.

Seed script seed.js that creates sample admin ad1/ad1, driver dr1/dr1, two students (22ME1A0501,22ME1A0502), a bus, and a sample route (geojson + two stops). Print sample JWT tokens and tripId(s) on seed run.

DEV_TESTS.md explaining local tests and a small socket-client-test.js script to simulate GPS pings.

Frontend:

React app with AuthContext storing token & user in sessionStorage and rehydrating on page refresh.

Axios api util that reads token from sessionStorage and sets Authorization header; 401 interceptor clears sessionStorage and redirects to login.

useSocket singleton hook: connects socket.io-client, emits auth:token on connect, reauths on token change, provides emitLocation(payload) that either sends or buffers when offline.

Admin Map Editor (/admin/routes) using Leaflet + leaflet.pm to draw route polyline and place stops; Save sends POST /api/routes with {name, geojson, stops}.

Driver Dashboard:

Start Trip API call (POST /api/trips/start) and receives tripId.

navigator.geolocation.watchPosition integration sending driver:location_update events via useSocket.emitLocation.

Client-side throttling to MIN_UPDATE_INTERVAL_MS, offline buffer, flush on reconnect.

Big buttons for Start Trip / Start Tracking / Stop Tracking / End Trip and status UI.

Driver simulator page (if running on desktop) to simulate a path and emit location updates.

Student Dashboard:

Map (Leaflet) showing student stop and moving bus marker from socket trip:location_update.

Shows ETA (prefer trip:eta_update; fallback compute from distance).

Notification opt-in toggle; use Notification API and service worker registration to show notifications both in-page and via SW (no VAPID).

Toast UI for in-page alerts.

Service Worker at /public/sw.js to handle showNotification and notification click.
